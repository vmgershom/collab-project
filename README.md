# Collab

Вебзастосунок для організації командної роботи студентів над навчальними проєктами з **автоматичним оцінюванням внеску кожного учасника** на основі методу аналізу ієрархій.

Викладач створює курси й проєкти, студенти приєднуються за кодом запрошення, формують команди, виконують завдання, спілкуються та оцінюють одне одного. Система за виміряними показниками (виконані завдання, своєчасність, активність, взаємні оцінки) розраховує частку внеску кожного студента в командний результат.

## Можливості

- Реєстрація та авторизація (ролі: студент / викладач), JWT.
- Курси й проєкти, запрошення за кодом, запис студентів на курс.
- Команди: створення, склад учасників, статус виконання.
- Завдання: призначення виконавцю, статуси, дедлайни, прикріплення файлів.
- Коментарі з гілками (загальна гілка команди та гілки окремих завдань).
- Взаємне оцінювання учасників команди (1–5).
- Журнал оцінок викладача та перегляд оцінок студентом.
- Календар дедлайнів проєктів і завдань (з урахуванням ролі).
- Навчальні матеріали курсу з файлами.
- Аналіз внеску учасників за методом Сааті.

## Технології

- **Бекенд:** Node.js, Express, Prisma ORM, PostgreSQL, JWT, bcryptjs, multer (завантаження файлів).
- **Фронтенд:** React, Vite, React Router.

## Вимоги

- Node.js (рекомендовано LTS) та npm.
- PostgreSQL.

## Встановлення та запуск

### 1. Клонувати репозиторій

```bash
git clone https://github.com/vmgershom/collab-project
cd collab-app
```

### 2. Підготувати базу даних

Створіть порожню базу даних PostgreSQL, наприклад `collab_db`.

### 3. Налаштувати бекенд

```bash
npm install
```

Створіть файл `.env` у корені проєкту за зразком `.env.example` і впишіть свої значення:

```env
DATABASE_URL="postgresql://postgres:ВАШ_ПАРОЛЬ@localhost:5432/collab_db"
JWT_SECRET="будь_який_секретний_рядок"
PORT=3000
```

Згенеруйте клієнт Prisma та застосуйте міграції (створять усі таблиці):

```bash
npx prisma generate
npx prisma migrate deploy
```

(Необов'язково) заповніть базу демонстраційними даними:

```bash
node prisma/seed.js
```

Запустіть бекенд (за замовчуванням на порту 3000):

```bash
node server.js
```

### 4. Налаштувати фронтенд

У новому терміналі:

```bash
cd client
npm install
npm run dev
```

Фронтенд запуститься за адресою `http://localhost:5173` і звертатиметься до бекенда на `http://localhost:3000`.

## Демонстраційні дані

Якщо ви виконали `node prisma/seed.js`, у базі будуть тестові користувачі (пароль у всіх — `test1234`):

- **Викладач:** `t1@test.com`
- **Студенти:** `s1@test.com` … `s8@test.com`

Сід створює два курси з проєктами, командами, виконаними завданнями, взаємними оцінками, коментарями та виставленими оцінками — зручно для перегляду готової картини.

> Увага: повторний запуск `node prisma/seed.js` повністю очищає базу перед заповненням.

## Структура проєкту

```
collab-app/
├─ prisma/
│  ├─ schema.prisma        # модель бази даних
│  ├─ migrations/          # міграції
│  └─ seed.js              # демонстраційні дані
├─ src/
│  ├─ routes/              # маршрути API
│  ├─ services/            # бізнес-логіка (зокрема розрахунок внеску)
│  ├─ utils/               # допоміжні модулі (AHP, доступи, матриця критеріїв)
│  ├─ middleware/          # автентифікація
│  ├─ prismaClient.js
│  └─ upload.js            # налаштування завантаження файлів
├─ server.js               # точка входу бекенда
└─ client/                 # фронтенд (React + Vite)
   └─ src/
      ├─ pages/            # сторінки
      ├─ components/       # компоненти
      ├─ api.js            # звернення до API
      └─ auth.js           # робота з токеном
```

## Примітки

- `node_modules`, файл `.env` та тека `uploads` навмисно не зберігаються в репозиторії: залежності відновлюються командою `npm install`, налаштування створюються з `.env.example`, а `uploads` наповнюється під час роботи застосунку.
- Розрахунок внеску реалізує метод аналізу ієрархій: ваги критеріїв задано експертною матрицею попарних порівнянь, а пріоритети студентів обчислюються нормуванням виміряних показників їхньої роботи в команді.
